### 第三步：查基線表（不能估算！）

```
5月1日=0%, 2日=1%, 3日=2%, 4日=4%, 5日=6%, 6日=9%, 7日=12%, 8日=16%, 9日=21%, 10日=26%

5月19日=52%, 20日=54%, 21日=57%, 22日=61%, 23日=66%, 24日=71%, 25日=74%, 26日=76%
5月27日=79%, 28日=82%, 29日=86%, 30日=92%, 31日=100%
```

### 第四步：計算今日需完成GMV

```
今日需完成GMV = 當月GMV目標 × 基線% - 當月GMV
```

**⚠️ 重要：係「目標×基線%-已完成」，不是「目標-已完成」！**

### 第五步：計算落後CC

- 落後條件：當月單量達成率 < 30%
- 從個人數據篩選

## 關鍵計算公式

| 公式 | 說明 |
|------|------|
| 今日需完成GMV = 當月GMV目標 × 基線% - 當月GMV | 正確公式 |
| 達成率 = 當月GMV ÷ 當月GMV目標 | GMV達成率 |
| 落後 = 單量達成率 < 30% | CC落後判定 |

## 錯誤預防

| ❌ 錯誤做法 | ✅ 正確做法 |
|------------|------------|
| 用「目標-GMV」計算今日需完成 | 必須用「目標×基線%-GMV」 |
| 估算基線 | 必須查表，5月11日=29%，5月12日=30% |
| 用上次的緩存數據 | 每次必須重新讀取Excel |
| 直接發送到群 | 必須先私發用戶確認 |

## 邊界條件（Fallback）

| 場景 | 處理 |
|------|------|
| Excel文件搵唔到 | 提示「益智業績表.xlsx未喺桌面搵到，請確認文件位置」 |
| Sheet名稱唔對 | 嘗試常見名稱：`2026年5月`、`5月`、`May` |
| 數據行位置有變 | 根據關鍵詞搜索行（如「03組合計」），唔 hardcode行號 |
| 落後CC名單為空 | 顯示「⚠️ 落後CC：無」，唔顯示空白 |
| GMV數值為負數 | 顯示「已超額完成」，唔顯示負數 |
| 數值為None或空 | 顯示「--」，唔顯示「None」 |
| 釘釘發送失敗 | 顯示錯誤信息，提示用戶手動複製消息內容 |
| 基線表未覆蓋該日 | 用線性插值估算，並喺消息中標註「（估算）」 |
