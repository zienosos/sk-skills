# CC深服注冊自動化技能

> 核心定位：用戶提供A+B → 全自動完成注冊 → 確認「成功」「已領」「恭喜」

---

## 決策樹（快速入口）

```
收到「注册CC深服」請求
    │
    ├── 有完整格式（A號碼 + B区号+號碼）
    │       └─→ 立即執行全流程（6步）
    │
    ├── 格式唔完整
    │       └─→ 問：「請提供格式：A號碼 / B区号 號碼」
    │
    ├── 問「深服係乜」「點解要注冊」
    │       └─→ 直接解釋：CC深服=家長免費領思維/美術體驗課的系統
    │
    └── 之前注冊過但失敗
            └─→ 先確認上一步係邊步失敗，再從該步繼續
```

## 觸發方式（必須確認格式）

用戶發「注册CC深服」並附格式：
```
A：號碼
B：区号 號碼
```
示例：
```
A：52931866
B：852 94198222
```

**⚠️ 格式必須完整才能開始，否則問清楚再繼續**

## 流程總覽（6步）

| 步驟 | 內容 | 成功標誌 |
|------|------|---------|
| 1 | 查詢平台用戶ID（CRM頁面） | 找到10位數字ID |
| 2 | 寫入飛書表格（平台用戶ID） | 寫入成功 |
| 3 | 打開注冊頁面（URL帶invite_userID） | 頁面打開 |
| 4 | 點擊「0元領取」紅色按鈕 | 按鈕響應 |
| 5 | 選區號 + 填電話 + 提交 | 提交成功 |
| 6 | 驗證「成功」「已領」「恭喜」 | 頁面出現關鍵詞 |

## 步驟1：查詢平台用戶ID

1. 連接Edge調試端口：`http://localhost:9222`
2. 導航到：`https://bizcenter-h5-cms.61info.cn/#/userManagement/userQuery`
3. 等待頁面加載**12秒**
4. 點擊左側菜單「用戶管理」（CDP，坐標約101,108）
5. 點擊「用戶查詢」（CDP）
6. 在表單裡：
   - 手機號下拉（已默認選中「手機號」）
   - Focus input[2]（y=101的輸入框）
   - 輸入A的號碼
7. 點擊「查詢」按鈕（CDP）
8. 從頁面文本匹配 `平台用戶id[：:\s]*(\d{10,})` 獲取10位數字

---

## 步驟2：寫入飛書表格

1. Wiki token: `AFIfwiufJiKqijk40SsciVVhnEb`
2. obj_token: `BCgDsFcoRh4G76tvx7zcFg4kn7W`（電子表格）
3. 用 `feishu_sheet` API寫入A列下一個空行
4. **⚠️ B列連結=號後面的號碼是「平台大用戶ID」**

---

## 步驟3：完成注冊

1. 打開B列生成的連結（URL帶invite_userID參數）
2. 點擊「0元領取思維美術大禮包」紅色按鈕（CDP原生點擊）
3. 表單彈出後：
   - **判斷區號**：
     - `+852` 或 `852` 開頭 → 選 **852**
     - `+853` 或 `853` 開頭 → 選 **853**
     - `+86` 或 `86` 開頭 → 選 **86**
   - 點擊區號下拉框 → 選對應區號（CDP原生點擊）
   - 在電話輸入框輸入B的號碼（唔係區號嗰個框！）
4. 點擊「0元立即免費領」提交（CDP原生點擊）
5. 驗證：頁面必須出現「成功」「已領」「恭喜」等字樣

## CDP原生鼠標事件（必須使用！）

```python
cdp = await page.context.new_cdp_session(page)
await cdp.send("Input.dispatchMouseEvent", {
    "type": "mouseMoved", "x": x, "y": y, "button": "none"
})
await cdp.send("Input.dispatchMouseEvent", {
    "type": "mousePressed", "x": x, "y": y, "button": "left", "clickCount": 1
})
await cdp.send("Input.dispatchMouseEvent", {
    "type": "mouseReleased", "x": x, "y": y, "button": "left", "clickCount": 1
})
```
其中x,y通過 `element.getBoundingClientRect()` 獲取。

**⚠️ 重要：這個Vue SPA不能用JS `.click()`！必須用CDP原生鼠標事件！**

## 邊界條件（Fallback）

| 場景 | 處理 |
|------|------|
| CRM頁面搵唔到用戶 | 告知用戶「搵唔到該號碼，請確認係咪已係平台用戶」 |
| 飛書表格寫入失敗 | 嘗試重新寫入；檢查token是否有效 |
| 注冊連結打唔開 | 檢查URL是否完整；更換瀏覽器重試 |
| 頁面搵唔到「0元領取」按鈕 | 等待頁面完全加載後重試；檢查是否已彈出表單 |
| 區號下拉選項不匹配 | 嚴格按B帳號格式判斷：+852→852，+853→853，+86→86 |
| 提交後頁面無「成功」字樣 | 等待3秒後再驗證；如仍無，嘗試重新填寫提交 |
| 連續操作失敗超過2次 | 告知用戶「操作遇到問題，請稍後重試或手動處理」 |
| 系統提示「您已購買同類商品」 | 該手機號之前已注冊過，告知用戶無需重複注冊 |
