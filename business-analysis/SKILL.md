# 業務分析（港澳CC專用）

> 核心定位：Excel數據 → 應用業務理解 → 診斷卡點在哪一步 → 明確團隊/個人維度 → 教練干預方向
> 與話術庫無關。與performance-report分工：前者=深度分析+教練方向；後者=純數據發送。

---

## 核心分析框架（每次分析必用）

### 漏斗七步診斷

```
例子數 → 分發數 → 約課數 → 到課數 → 成交數
   ↓        ↓        ↓        ↓        ↓
流量問題  分發問題  約課問題  應到課問題  成交/報價問題
```

### 完整指標體系（必須全部走完）

| 指標 | 目標 | 教練意義 |
|------|------|---------|
| 分發率 | 越高越好 | CC拿到多少資源 |
| 分發約課率 | ≥40% | CC跟進節奏 |
| 應到課率 | ≥75% | CC課前確認動作 |
| 約課到課率 | ≥80% | 家長意願+CC跟進 |
| 到課轉化率 | ≥25% | 課後逼單能力 |
| DCP率 | ≥20% | 當場逼單執行情況 |
| 平均成交周期 | 越低越好 | 考慮時間過長的信號 |
| ASP | 4000-14000 | 報價策略是否合理 |
| 產能達成率 | ≥100% | CC整體產出能力 |

### 分析順序（不能跳步）

1. **先看漏斗整體**：哪一步掉得最多？
2. **再看過程指標**：對照目標閾值，確認是哪個環節異常
3. **結合業務語境**：結合渠道特性（KOL/投放/商超/轉介紹）、月份規律（書展大月/淡季）
4. **團隊維度 → 個人維度**：先確認是小組問題還是個人問題
5. **應用教練方向**：DCP+轉化組合只是教練方向快速入口，不是全部

### DCP率+轉化率組合判斷（教練方向快速入口）

| DCP率 | 轉化率 | 問題所在 | 教練方向 |
|-------|--------|---------|---------|
| 高 | 低 | 回單量不足 | 查客戶跟進頻次 + 客包推送策略 |
| 低 | 高 | 動力不足 | 核查KPI設定 + 逼單意願 |
| 高 | 高 | 理想狀態 | 維持現況 |
| 低 | 低 | 綜合問題 | 全面核查：意願+能力+策略 |

### 團隊維度 vs 個人維度

**先看團隊，再看個人：**
1. 團隊落後 → 是全員問題還是少數人拉低？
2. 定位到個人 → 確認教練方向

## 步驟2：解析Excel（XML底層法）

**⚠️ 警告：openpyxl直接讀中文會乱码，必須用zipfile+XML**

```python
import zipfile, xml.etree.ElementTree as ET

def parse_xlsx(path):
    """返回 (strings列表, rows二維數組)"""
    with zipfile.ZipFile(path) as z:
        with z.open('xl/sharedStrings.xml') as f:
            tree = ET.parse(f)
        ns = {'ns': 'http://schemas.openxmlformats.org/spreadsheetml/2006/main'}
        strings = []
        for si in tree.findall('.//ns:si', ns):
            t = ''.join(tt.text or '' for tt in si.findall('.//ns:t', ns))
            strings.append(t)

        with z.open('xl/worksheets/sheet1.xml') as f:
            tree = ET.parse(f)

        rows = []
        for row in tree.findall('.//ns:row', ns):
            vals = []
            for c in row.findall('ns:c', ns):
                typ = c.get('t', '')
                v = c.find('ns:v', ns)
                if v is not None:
                    if typ == 's':
                        vals.append(strings[int(v.text)])
                    else:
                        vals.append(v.text)
                else:
                    vals.append('')
            rows.append(vals)
        return strings, rows
```

## 步驟3：應用渠道路徑框架

### HK_BIZ欄位速查（關鍵閾值）

| 列 | 欄位 | 目標閾值 |
|----|------|---------|
| C | 分發數 | — |
| D | 分發轉化率 | — |
| F | 分發約課率 | 目標≥40% |
| N | 應到課率 | 目標≥75% |
| O | 約課到課率 | 目標≥80% |
| Q | 到課轉化率（到轉） | 目標≥25% |
| S | DCP率 | 目標≥20% |
| T | 平均成交周期 | 越低越好 |
| V | ASP | 正常4000-14000 |
| X | 產能達成率 | 目標100% |

### 問題定位決策樹

```
① 分發轉化率低？
  ├─ YES → ② 到課率正常？
  │         ├─ YES → 報價/話術問題 → 推小課包（注意：4388並非每日上架，按活動安排）
  │         └─ NO  → 應到課率問題 → 查CC課前24H確認動作
  └─ NO  → ③ 分發約課率低？
            ├─ YES → CC跟進節奏問題
            └─ NO  → 已約課但未到 → 課前2H提醒缺失
```

## 邊界條件

| 場景 | 處理 |
|------|------|
| Excel編碼乱码 | 切換XML底層解析 |
| 分發轉化率>50% | 正常，着重分析到轉維度 |
| 分發轉化率<5% | 緊急：立即分流+小課包策略（視活動情況）|
| 飛書寫入失敗 | 顯示原文摘要，提示用戶手動複製 |
